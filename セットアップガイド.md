# 議事録練習アプリ セットアップガイド(完全無料・初心者向け)

プログラミングの知識は不要です。ブラウザだけで完結します。所要時間は30分ほど。
作るアカウントは4つ: **GitHub**(コード置き場)、**Supabase**(データベース)、**Google AI Studio**(AI・無料)、**Vercel**(公開サーバー)。すべて無料です。

---

## STEP 1. GitHubにコードを置く(約10分)

1. https://github.com を開き「Sign up」で無料登録(メールアドレスがあればOK)
2. ログイン後、右上の「+」→「New repository」
3. Repository name に `minutes-trainer` と入力 →「Private」を選択 →「Create repository」
4. できたページの「uploading an existing file」というリンクをクリック
5. パソコンの `minutes-trainer` フォルダを開き、**中身のファイル・フォルダを全部選択**してドラッグ&ドロップ
   - ⚠️ `minutes-trainer` フォルダごとではなく「中身」をドロップしてください
   - ⚠️ `node_modules` フォルダがもしあれば、それだけは除外してください
   - フォルダ(app、lib、components)ごとドラッグすれば中のファイルもアップロードされます
6. 下の「Commit changes」ボタンを押す

※うまくドラッグできない場合は「Add file → Upload files」から複数回に分けてアップロードしてもOKです。

## STEP 2. データベースを作る(Supabase・約7分)

1. https://supabase.com →「Start your project」→ GitHubアカウントでログイン
2. 「New project」→ 以下を入力して「Create new project」
   - Name: `minutes-trainer`
   - Database Password: 適当な強いパスワード(**必ずメモ**)
   - Region: `Northeast Asia (Tokyo)`
3. 1〜2分待つとプロジェクトが起動
4. 左メニューの「SQL Editor」→「New query」→ パソコンの `schema.sql` をメモ帳で開いて中身を全部コピー&貼り付け →「Run」
   - 「Success. No rows returned」と出ればOK
5. 画面上部の「Connect」ボタン → 「Connection string」タブ → **Transaction pooler** の URI をコピー
   - `postgresql://postgres.xxxx:[YOUR-PASSWORD]@aws-0-ap-northeast-1...` のような文字列
   - `[YOUR-PASSWORD]` の部分を手順2でメモしたパスワードに置き換える(括弧 [ ] も消す)
   - これが `DATABASE_URL` です。メモ帳に控えておく

## STEP 3. AIの無料APIキーを作る(Google AI Studio・約3分)

1. https://aistudio.google.com を開き、Googleアカウントでログイン
2. 「Get API key」→「Create API key」
3. 表示されたキー(`AIza...` で始まる文字列)をコピーしてメモ帳に控える
   - これが `GEMINI_API_KEY` です
   - 無料枠内で使うのでクレジットカード登録は不要です

## STEP 4. 公開する(Vercel・約7分)

1. https://vercel.com →「Sign Up」→「Continue with GitHub」でログイン
2. 「Add New...」→「Project」→ `minutes-trainer` リポジトリの「Import」
3. 「Environment Variables」を開いて、以下を1行ずつ追加(Name / Value):

   | Name | Value |
   |---|---|
   | `DATABASE_URL` | STEP 2で控えた接続文字列 |
   | `GEMINI_API_KEY` | STEP 3で控えたキー |
   | `JWT_SECRET` | 適当な長いランダム文字列(例: キーボードを50文字めちゃくちゃに打つ) |

4. 「Deploy」を押して2〜3分待つ
5. 「Congratulations!」と出たら完成。表示されたURL(`https://minutes-trainer-xxxx.vercel.app`)があなたのアプリです

## STEP 5. 動作確認

1. 発行されたURLを開く → 「新規登録」でアカウント作成
   - **最初に登録した人が自動で管理者**になります(全員の成績を見られる)
2. 「新しい練習」→ テーマ等を選んで「会議を生成する」(30秒ほど)
3. 会議を再生(音声が出ない場合はTTSチェックを確認。Chrome/Edge推奨)
4. 再生終了後、Wordで議事録を作成 → 「PDF形式で保存」 → アップロードして提出
5. 採点結果が表示されれば成功。URLを他のメンバーに共有すれば全員使えます

## よくあるつまずき

| 症状 | 対処 |
|---|---|
| デプロイでビルドエラー | GitHubに `package.json` がリポジトリの直下にあるか確認(フォルダごとアップしていると1階層深くなる)。その場合はVercelのプロジェクト設定 → Root Directory を `minutes-trainer` に変更 |
| 「サーバーエラー」が出る | `DATABASE_URL` のパスワード置き換え忘れ・打ち間違いが最多。Vercelの Settings → Environment Variables で修正 → Deployments → 最新のものを「Redeploy」 |
| 会議生成で失敗する | `GEMINI_API_KEY` の値を確認。無料枠の分あたり回数制限に当たった場合は1分待って再試行 |
| PDF提出でエラー | スキャン画像のPDFは不可。WordやGoogleドキュメントから書き出したPDFを使用。4MB以下 |
| 音声が読み上げられない | Chrome/Edgeを使用。macOSはシステム設定で日本語音声のダウンロードが必要な場合あり |

## 無料枠の目安

- Gemini無料枠: 1日あたり十分な回数(数百リクエスト)。研修利用(〜10名)なら通常収まります
- Supabase無料枠: DB 500MB。提出PDF数千件分
- Vercel無料枠: 個人・小規模利用なら十分
- アプリ側でも1人1日「生成5回・採点10回」の制限が入っています(Vercelの環境変数 `DAILY_GENERATE_LIMIT` / `DAILY_SCORE_LIMIT` で変更可)

## アプリを更新したいとき

GitHubのリポジトリでファイルを差し替えて Commit すると、Vercelが自動で再デプロイします。
